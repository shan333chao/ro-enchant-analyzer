# OCR 引擎说明

## 概述

OCR 引擎负责识别附魔截图中的文字信息，提取属性名称、属性数值和组合特性。

## 技术选型

- **引擎**：OnnxOCR（PP-OCRv5 ONNX 模型）
- **推理**：onnxruntime（CPU 模式）
- **图像处理**：OpenCV + numpy
- **识别速度**：约 0.5 秒/张（含 HTTP 开销）

## 识别流程

```
输入图片
  ↓
预处理 (_preprocess_image)
  ├─ 大图(≥800x400): 不做预处理，直接OCR
  └─ 小图(<800x400): 4x放大 + CLAHE灰度增强
  ↓
双通OCR
  ├─ 彩色版 OCR
  └─ CLAHE灰度版 OCR
  ↓
合并去重 (位置相近的保留高置信度)
  ↓
固定布局OCR (_ocr_fixed_layout) ← 仅501x185小图
  ├─ 按行裁剪(3行属性 + 1行特性)
  ├─ 每块8x放大 + CLAHE
  └─ 单独OCR后拼接
  ↓
垃圾文本过滤
  ├─ 单字符(非数字) → 过滤
  ├─ 水印/UI符号(©, □等) → 过滤
  └─ 纯符号串 → 过滤
  ↓
按行分组排序
  ├─ cy聚类(30px阈值)分行
  └─ 行内cx左到右排序
  ↓
属性解析 (parse_attributes)
  ├─ OCR别名纠正 (名引→名弓 等)
  ├─ 歧义名消歧 (根据值范围判断)
  ├─ 属性名-数值配对
  └─ 组合特性解析
  ↓
输出结果
```

## 预处理参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 放大倍数 | 4x (INTER_CUBIC) | 小图文字太小，放大提高识别率 |
| CLAHE clipLimit | 2.0 (通用) / 4.0 (固定布局) | 对比度增强 |
| CLAHE tileGridSize | 8x8 | 局部对比度窗口 |

## OnnxOCR 参数

| 参数 | 值 | 说明 |
|------|-----|------|
| det_db_thresh | 0.2 | 文本检测阈值（降低以检测更多文本） |
| det_db_box_thresh | 0.3 | 文本框置信度阈值 |
| det_db_unclip_ratio | 2.0 | 文本框扩展比例 |
| use_dilation | True | 膨胀操作增强细小文本检测 |
| use_angle_cls | True | 文字方向分类 |
| use_gpu | False | CPU 模式 |

## OCR 别名纠正

OCR 可能产生的常见识别错误，系统自动纠正：

| OCR 识别文本 | 纠正为 | 类型 |
|-------------|--------|------|
| 名引/名利/名刃 | 名弓 | 特性名 |
| 褻读/亵读/亵渎 | 褻渎 | 特性名 |
| 物里防御 | 物理防御 | 属性名 |
| 魔决防御/魔法防卸 | 魔法防御 | 属性名 |
| 暴伤% | 爆伤% | 属性名 |
| 各种「扺抗」 | 对应「抵抗」 | 属性名 |
| 幸邉 | 幸运 | 属性名 |

## 歧义消歧

部分 OCR 识别结果可能对应多个不同属性，系统根据数值范围判断：

| OCR 文本 | 候选属性 | 判定规则 |
|----------|---------|---------|
| 母侣 | 命中(5~30) / 幸运(4~8) | 值<=4→幸运，否则→命中 |

## 固定布局识别

针对 501x185 尺寸的附魔截图（最常见尺寸），使用固定布局裁剪方式提高识别精度：

```
图片布局 (501x185):
┌─────────────────────────────┐
│ Row1: [属性名]    [属性值]   │ (5~40px)
│ Row2: [属性名]    [属性值]   │ (42~77px)
│ Row3: [属性名]    [属性值]   │ (79~114px)
│ Row4: [组合特性]             │ (116~165px)
└─────────────────────────────┘
```

每块裁剪后 8x 放大 + CLAHE 增强再 OCR，识别率显著提升。

## 日志

每次 OCR 识别会记录日志到 `logs/ocr/` 目录：
- `{log_id}.json` - 完整识别结果（含原始文本和解析后数据）
- `ocr_log.txt` - 文本格式追加日志
